# Test App (iOS)

## Описание

Простое iOS-приложение, реализующее загрузку веб-контента через `WKWebView`.
Приложение открывает заданный URL сразу при запуске без промежуточных экранов.

---

## 🛠 Стек технологий

* **SwiftUI** — UI слой
* **WKWebView** — отображение веб-контента
* **UIKit (UIViewRepresentable)** — интеграция WebView в SwiftUI
* **UserNotifications** — работа с push-разрешениями
* **PhotosUI** — загрузка изображений через WebView
* **UserDefaults** — хранение флагов (например, запроса уведомлений)

## Принятые решения

### ✔ Web-only архитектура

Отказ от нативных экранов и конфигурационных API в пользу максимально простой и быстрой загрузки контента в соответствии с ТЗ.

---

### ✔ Контроль push-запроса

Реализован кастомный флаг, чтобы избежать повторных системных запросов.

---

### ✔ Anti-detect меры

Добавлены JS-патчи для маскировки WebView:

* `navigator.webdriver`
* `navigator.platform`
* `navigator.languages`
* `window.chrome`

---

### ✔ Обработка редиректов

Добавлен retry-механизм при `NSURLErrorHTTPTooManyRedirects`.

---

## ⚠️ Ограничения в данном патче

* URL захардкожен (нет remote config)
* отсутствует обработка оффлайн-сценариев
* нет fallback UI при ошибках загрузки (рассчитываем, что загрузка будет в любом случае)
* push-уведомления только запрашиваются (без интеграции SDK)
